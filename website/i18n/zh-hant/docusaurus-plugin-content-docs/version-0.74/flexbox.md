元件可以透過 Flexbox 演算法來指定其子元素的佈局方式。Flexbox 旨在為不同螢幕尺寸提供一致的佈局。

通常會結合使用 `flexDirection`、`alignItems` 和 `justifyContent` 來達成正確的佈局。

:::caution
Flexbox 在 React Native 中的運作方式與網頁 CSS 相同，但有幾點例外：
預設值不同，`flexDirection` 預設為 `column` 而非 `row`，`alignContent` 預設為 `flex-start` 而非 `stretch`，`flexShrink` 預設為 `0` 而非 `1`，且 `flex` 參數僅支援單一數值。
:::

## Flex 彈性佈局

[`flex`](layout-props#flex) 屬性定義子元素如何沿主軸「填滿」可用空間。空間將根據各元素的 flex 屬性比例分配。

在以下範例中，紅色、橘色和綠色視圖都是設有 `flex: 1` 的容器視圖的子元素。紅色視圖使用 `flex: 1`，橘色視圖使用 `flex: 2`，綠色視圖使用 `flex: 3`。**1+2+3 = 6**，這表示紅色視圖將獲得 `1/6` 的空間，橘色視圖獲得 `2/6`，綠色視圖獲得 `3/6`。

```SnackPlayer name=Flex%20Example
import React from 'react';
import {StyleSheet, View} from 'react-native';

const Flex = () => {
  return (
    <View
      style={[
        styles.container,
        {
          // Try setting `flexDirection` to `"row"`.
          flexDirection: 'column',
        },
      ]}>
      <View style={{flex: 1, backgroundColor: 'red'}} />
      <View style={{flex: 2, backgroundColor: 'darkorange'}} />
      <View style={{flex: 3, backgroundColor: 'green'}} />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
  },
});

export default Flex;
```

## 主軸方向 (Flex Direction)

[`flexDirection`](layout-props#flexdirection) 控制子節點的排列方向（也稱為主軸）。交叉軸則是與主軸垂直的方向，或是換行時的排列方向。

- `column`（**預設值**）從上到下排列子元素。若啟用換行，則下一行會從容器頂部第一個項目的右側開始。
- `row` 從左到右排列子元素。若啟用換行，則下一行會從容器左側第一個項目的下方開始。
- `column-reverse` 從下到上排列子元素。若啟用換行，則下一行會從容器底部第一個項目的右側開始。
- `row-reverse` 從右到左排列子元素。若啟用換行，則下一行會從容器右側第一個項目的下方開始。

可參考[此連結](https://www.yogalayout.dev/docs/styling/flex-direction)了解更多。

  label,
  children,
  values,
  selectedValue,
  setSelectedValue,

    <View style={styles.row}>

          key={value}
          onPress={() => setSelectedValue(value)}

            {value}
          </Text>

    marginTop: 8,
    backgroundColor: 'aliceblue',
  },
  box: {
    width: 50,
    height: 50,
  },
  row: {

    flexWrap: 'wrap',
  },

## 佈局方向 (Layout Direction)

佈局 [`direction`](layout-props#direction) 指定子元素和文字在層級結構中的排列方向，同時影響 `start` 和 `end` 所指的邊緣。React Native 預設採用 LTR（左至右）佈局方向，此時 `start` 指左側，`end` 指右側。

- `LTR`（**預設值**）文字和子元素從左至右排列。應用於元素起始端的邊距和內距會作用於左側。
- `RTL` 文字和子元素從右至左排列。應用於元素起始端的邊距和內距會作用於右側。

## 主軸對齊 (Justify Content)

[`justifyContent`](layout-props#justifycontent) 描述如何沿容器主軸對齊子元素。例如：當 `flexDirection` 設為 `row` 時可水平置中子元素，設為 `column` 時可垂直置中。

- `flex-start`（**預設值**）將容器的子元素對齊至主軸起始端。
- `flex-end` 將容器的子元素對齊至主軸末端。
- `center` 將容器的子元素對齊至主軸中心。
- `space-between` 沿主軸均勻分布子元素，剩餘空間分配於子元素之間。
- `space-around` 沿主軸均勻分布子元素，剩餘空間環繞子元素分配（相較於`space-between`，首尾子元素外側也會分配空間）。
- `space-evenly` 沿主軸完全均等分布子元素，相鄰元素間距、首元素與容器起始端間距、末元素與容器末端間距完全一致。

詳細說明請參閱[此文件](https://www.yogalayout.dev/docs/styling/justify-content)。

## 對齊項目（Align Items）

[`alignItems`](layout-props#alignitems)控制子元素在容器交叉軸上的對齊方式，其作用原理與`justifyContent`相似，但作用於交叉軸而非主軸。

- `stretch`（**預設值**）拉伸子元素高度以填滿容器交叉軸。
- `flex-start` 將子元素對齊至交叉軸起始端。
- `flex-end` 將子元素對齊至交叉軸末端。
- `center` 將子元素對齊至交叉軸中心。
- `baseline` 使子元素沿基線對齊，各子元素可設為父元素的基準線。

:::info
要使`stretch`生效，子元素在次軸上不能有固定尺寸。下例中，除非移除子元素的`width: 50`設定，否則`alignItems: stretch`不會產生效果。
:::

進階說明請參閱[此文件](https://www.yogalayout.dev/docs/styling/align-items-self)。

## 對齊內容（Align Content）

[alignContent](layout-props#aligncontent)定義多行元素在交叉軸上的分布方式，僅在啟用`flexWrap`換行時有效。

- `flex-start`（**預設值**）將換行內容對齊至交叉軸起始端。
- `flex-end` 將換行內容對齊至交叉軸末端。
- `stretch`（網頁版Yoga預設值）拉伸換行內容高度以填滿交叉軸。
- `center` 將換行內容對齊至交叉軸中心。
- `space-between` 沿交叉軸均勻分布換行內容，剩餘空間分配於行間。
- `space-around` 沿交叉軸均勻分布換行內容，剩餘空間環繞每行分配（容器兩端空間為行間距的一半）。
- `space-evenly` 沿交叉軸完全均等分布換行內容，所有間距大小一致。

深入學習請參閱[此文件](https://www.yogalayout.dev/docs/styling/align-content)。

## Flex Basis、Grow 與 Shrink

- [`flexBasis`](layout-props#flexbasis) 是一種與軸向無關的方式，用於設定項目沿主軸的預設尺寸。設定子元素的 `flexBasis` 類似於在父容器為 `flexDirection: row` 時設定該子元素的 `width`，或在父容器為 `flexDirection: column` 時設定其 `height`。項目的 `flexBasis` 是該項目的預設尺寸，即在進行任何 `flexGrow` 和 `flexShrink` 計算之前的尺寸。

- [`flexGrow`](layout-props#flexgrow) 描述容器內應如何沿主軸分配剩餘空間給其子元素。在佈局其子元素後，容器會根據子元素指定的 flex grow 值分配任何剩餘空間。

  `flexGrow` 接受任何大於或等於 0 的浮點數，預設值為 0。容器會根據子元素的 `flexGrow` 值加權分配剩餘空間。

- [`flexShrink`](layout-props#flexshrink) 描述當子元素的總尺寸在主軸上溢出容器尺寸時，應如何沿主軸收縮子元素。`flexShrink` 與 `flexGrow` 非常相似，可以將任何溢出的尺寸視為負的剩餘空間來理解。這兩個屬性也能很好地協同工作，允許子元素根據需要伸展或收縮。

  `flexShrink` 接受任何大於或等於 0 的浮點數，預設值為 0（在網頁上，預設值為 1）。容器會根據子元素的 `flexShrink` 值加權收縮子元素。

了解更多[請參閱此處](https://www.yogalayout.dev/docs/styling/flex-basis-grow-shrink)。

## 寬度與高度

`width` 屬性指定元素內容區域的寬度。類似地，`height` 屬性指定元素內容區域的高度。

`width` 和 `height` 均可接受以下值：

- `auto`（**預設值**）React Native 會根據元素的內容（無論是其他子元素、文字還是圖片）計算其寬度/高度。

- `pixels` 以絕對像素定義寬度/高度。根據組件上設置的其他樣式，這可能並非節點的最終尺寸。

- `percentage` 以父元素寬度或高度的百分比分別定義寬度或高度。

## 絕對與相對佈局

元素的 `position` 類型定義了其在父元素內的定位方式。

- `relative` (**預設值**) 預設情況下，元素採用相對定位。這表示元素會依照正常的版面配置流程進行定位，然後根據 `top`、`right`、`bottom` 和 `left` 的值進行相對偏移。此偏移不會影響任何兄弟元素或父元素的位置。

- `absolute` 當採用絕對定位時，元素不會參與正常的版面配置流程。而是獨立於其兄弟元素進行佈局。其位置由 `top`、`right`、`bottom` 和 `left` 的值決定。
